# Part 4. UI/UX 설계

## 4.1 디자인 원칙 (7대 원칙)

### Typora 5대 원칙 계승 + 2025 트렌드 2대 추가

Baram의 UI/UX 설계는 7개의 명시적 원칙에 기반한다. 처음 5개는 Typora에서 증명된 마크다운 WYSIWYG의 핵심이며, 나머지 2개는 2025년 에디터 트렌드에서 도출했다.

| # | 원칙 | 정의 | 구현 핵심 |
|---|------|------|----------|
| 1 | **구문 불가시성** (Syntax Invisibility) | 포커스 없는 마크다운 구문은 렌더링 결과로 대체된다 | InputRule + Decoration 포커스/블러 전환 |
| 2 | **즉각 반응** (Instant Feedback) | 타이핑 즉시 렌더링, 키 입력 → 화면 업데이트 16ms 이하 | ProseMirror 트랜잭션 최적화, 디바운스 최소화 |
| 3 | **최소 인터페이스** (Minimal Interface) | 기본 상태에서 에디터 영역만 보인다 | 사이드바 기본 숨김, 메뉴바 최소화, 크롬 제거 |
| 4 | **마크다운 충실도** (Markdown Fidelity) | 출력은 항상 클린한 표준 마크다운 파일이다 | 커스텀 Serializer, Roundtrip Fidelity 보장 |
| 5 | **점진적 복잡도** (Progressive Complexity) | 기본 기능은 즉시, 고급 기능은 필요할 때 활성화 | Extension 토글, 확장 문법 설정 |
| 6 | **Command-First** | 커맨드 팔레트가 모든 기능의 단일 진입점이다 | Cmd+K 퍼지 검색, 메뉴바 → 커맨드 팔레트 |
| 7 | **Contextual Floating UI** | 필요한 UI가 맥락에 맞는 위치에 자동으로 나타난다 | 플로팅 서식 툴바, 블록 핸들, 호버 버튼 |

원칙 간 관계: 원칙 3(최소 인터페이스)과 원칙 5(점진적 복잡도)는 "기본 상태는 단순하게"라는 공통 방향이고, 원칙 6(Command-First)과 원칙 7(Contextual Floating UI)은 "그럼에도 모든 기능에 빠르게 접근"하는 해법이다.

### 디자인 시스템

#### 색상 체계

CSS Variables 기반으로 모든 색상을 정의하여 테마 전환 시 변수만 교체한다.

```css
/* Light Theme */
:root {
  /* 배경 */
  --bg-primary: #ffffff;           /* 에디터 배경 */
  --bg-secondary: #f8f9fa;         /* 사이드바 배경 */
  --bg-tertiary: #f0f1f3;          /* 호버, 선택 배경 */
  --bg-elevated: #ffffff;          /* 모달, 드롭다운 */

  /* 텍스트 */
  --text-primary: #1a1a1a;         /* 본문 텍스트 */
  --text-secondary: #6b7280;       /* 보조 텍스트, 플레이스홀더 */
  --text-tertiary: #9ca3af;        /* 비활성 텍스트 */
  --text-link: #2563eb;            /* 링크 */

  /* 구문 요소 */
  --syntax-heading: #1a1a1a;       /* 헤딩 텍스트 */
  --syntax-bold: #1a1a1a;          /* 볼드 */
  --syntax-code-bg: #f3f4f6;       /* 인라인 코드 배경 */
  --syntax-code-text: #e11d48;     /* 인라인 코드 텍스트 */
  --syntax-blockquote: #6b7280;    /* 인용 텍스트 */
  --syntax-highlight-bg: #fef08a;  /* 하이라이트 배경 */
  --syntax-math: #7c3aed;          /* 수식 */
  --syntax-wikilink: #2563eb;      /* [[위키링크]] */
  --syntax-ghost: #d1d5db;         /* AI Ghost Text */

  /* 액센트 */
  --accent-primary: #2563eb;       /* 메인 액센트 (링크, 선택) */
  --accent-ai: #8b5cf6;            /* AI 관련 요소 */
  --accent-success: #10b981;       /* 성공, diff 추가 */
  --accent-warning: #f59e0b;       /* 경고 */
  --accent-danger: #ef4444;        /* 삭제, 오류, diff 삭제 */

  /* 테두리 */
  --border-primary: #e5e7eb;       /* 기본 테두리 */
  --border-focus: #2563eb;         /* 포커스 테두리 */

  /* 그림자 */
  --shadow-sm: 0 1px 2px rgba(0,0,0,0.05);
  --shadow-md: 0 4px 6px rgba(0,0,0,0.07);
  --shadow-lg: 0 10px 25px rgba(0,0,0,0.1);
  --shadow-floating: 0 4px 12px rgba(0,0,0,0.12);  /* 플로팅 UI */
}

/* Dark Theme */
[data-theme="dark"] {
  --bg-primary: #1e1e1e;
  --bg-secondary: #252526;
  --bg-tertiary: #2d2d2d;
  --bg-elevated: #333333;
  --text-primary: #e0e0e0;
  --text-secondary: #a0a0a0;
  --text-tertiary: #666666;
  --text-link: #60a5fa;
  /* ... 나머지 dark 매핑 */
}
```

#### 타이포그래피

| 용도 | 폰트 | 크기 | 줄 높이 |
|------|------|------|---------|
| 에디터 본문 | 시스템 산세리프 (기본) / 사용자 선택 | 16px (기본, 12~24 조절) | 1.8 |
| H1 | 에디터 본문 | 2.0em (32px) | 1.3 |
| H2 | 에디터 본문 | 1.5em (24px) | 1.4 |
| H3 | 에디터 본문 | 1.25em (20px) | 1.5 |
| H4~H6 | 에디터 본문 | 1.0em (16px), 굵기/스타일로 구분 | 1.6 |
| 인라인 코드 | 모노스페이스 | 0.9em | — |
| 코드 블록 | 모노스페이스 | 14px | 1.6 |
| UI 요소 | 시스템 UI 폰트 | 13px | 1.4 |
| 상태바 | 시스템 UI 폰트 | 12px | 1.0 |

에디터 본문의 최대 폭은 기본 720px로 제한하여 읽기 편한 줄 길이를 유지한다. 설정에서 640~960px 범위로 조절 가능하며, "무제한" 옵션도 제공한다.

#### 간격 체계

8px 그리드 기반. `--spacing-1: 4px`, `--spacing-2: 8px`, `--spacing-3: 12px`, `--spacing-4: 16px`, `--spacing-6: 24px`, `--spacing-8: 32px`. 에디터 좌우 패딩 `--spacing-8`(32px), 블록 간 간격 `--spacing-4`(16px), 헤딩 상단 여백 `--spacing-6`(24px).

#### 다크/라이트 모드 전략

1차: 시스템 설정 자동 감지(`prefers-color-scheme`). 2차: 커맨드 팔레트에서 "theme" 검색하여 수동 전환. 3차: 설정에서 고정(Light / Dark / System Auto). 테마 전환 시 CSS Variables 값만 교체하며, 전환 애니메이션(200ms ease)을 적용한다.

## 4.2 3-Layer Interaction 모델

전통적인 메뉴바 중심 인터페이스 대신 3계층 인터랙션 모델을 채택한다. 이 모델은 "사용자와 기능 사이의 거리"를 기준으로 모든 상호작용을 분류한다.

```
┌──────────────────────────────────────────────────────┐
│                                                      │
│   Layer 1: Inline      타이핑 흐름 안에서              │
│   ─────────────────    거리 = 0 (키보드에서 손 안 뗌)  │
│   /  [[  ((  $  @  Tab  마크다운 구문                  │
│                                                      │
├──────────────────────────────────────────────────────┤
│                                                      │
│   Layer 2: Contextual  맥락에 따라 자동 출현            │
│   ─────────────────    거리 = 1 (클릭 또는 호버)       │
│   플로팅 툴바  블록 핸들  호버 버튼  우클릭 메뉴         │
│   Inline Edit(Cmd+K)                                 │
│                                                      │
├──────────────────────────────────────────────────────┤
│                                                      │
│   Layer 3: Global      앱 전체에서 호출                │
│   ─────────────────    거리 = 2 (단축키로 진입점 호출)  │
│   Cmd+K(커맨드)  Cmd+O(파일)  Cmd+Shift+F(검색)      │
│   Cmd+Shift+A(AI)  Cmd+,(설정)                       │
│                                                      │
└──────────────────────────────────────────────────────┘
```

### 4.2.1 Layer 1: Inline — 타이핑 흐름 안에서

Layer 1은 사용자가 키보드에서 손을 떼지 않고 에디터 내에서 직접 트리거하는 상호작용이다.

**슬래시 커맨드 (`/`)**

빈 줄 또는 줄 시작에서 `/`를 입력하면 블록 삽입 메뉴가 나타난다.

```
│ (빈 줄에서 / 입력)
│
│ /head                              ← 타이핑하면 퍼지 필터
│ ┌──────────────────────────────┐
│ │ 📝 Heading 1           # 제목 │   ← 마크다운 힌트 표시
│ │ 📝 Heading 2          ## 제목 │
│ │ 📝 Heading 3         ### 제목 │
│ │ ── 기본 ──────────────────── │
│ │ 📋 Table                Ctrl+T│
│ │ 💻 Code Block        ``` lang│
│ │ 📐 Math Block           $$   │
│ │ 🎨 Mermaid Diagram           │
│ │ 📌 Callout           > [!tip]│
│ │ ── AI ───────────────────── │
│ │ ✨ AI로 초안 작성              │
│ │ ✨ AI 요약                     │
│ │ ✨ AI 번역                     │
│ └──────────────────────────────┘
```

설계 핵심: 슬래시 커맨드와 마크다운 구문은 **공존**한다. 마크다운을 아는 사용자는 `# `을 직접 타이핑하고, 모르는 사용자는 `/heading`으로 같은 결과를 얻는다. 슬래시 메뉴의 각 항목 오른쪽에 마크다운 구문 힌트를 표시하여 자연스러운 학습을 유도한다.

**Wikilink 자동완성 (`[[`)**

`[[`를 입력하면 Vault 내 파일 목록이 퍼지 검색 팝업으로 나타난다.

```
│ 이 내용은 [[ski              ← [[ 입력 후 타이핑
│ ┌──────────────────────────┐
│ │ 📄 skill-a.md             │
│ │ 📄 skill-b.md             │
│ │ 📄 skills-overview.md     │
│ │ ─────────────────────── │
│ │ + 새 파일 "ski" 생성       │  ← 결과 없으면 즉시 생성 옵션
│ └──────────────────────────┘
```

별칭(aliases) 매칭을 지원하여, frontmatter에 `aliases: [AI 스킬]`이 정의되어 있으면 `[[AI 스킬`로도 검색된다. `[[파일명#헤딩]]`으로 특정 헤딩에 링크하거나, `[[파일명^block-id]]`로 블록에 링크할 수 있다.

**블록 참조 자동완성 (`((`)**

`((`를 입력하면 Vault 내 모든 블록을 검색하는 팝업이 나타난다. 블록의 첫 줄 텍스트가 미리보기로 표시된다.

**마크다운 구문 즉시 변환**

Typora의 핵심 패턴을 그대로 계승한다.

| 입력 | 트리거 시점 | 결과 |
|------|-----------|------|
| `# ` + 텍스트 | Space 입력 시 | H1으로 변환 |
| `## ` ~ `###### ` | Space 입력 시 | H2~H6으로 변환 |
| `> ` | Space 입력 시 | 인용 블록으로 변환 |
| `- ` / `* ` / `1. ` | Space 입력 시 | 리스트로 변환 |
| `- [ ] ` | Space 입력 시 | 태스크 리스트로 변환 |
| `---` | Enter 입력 시 | 수평선으로 변환 |
| ```` ``` ```` + 언어 | Enter 입력 시 | 코드 블록 생성 |
| `$$` | Enter 입력 시 | 수식 블록 생성 |
| `**text**` | 닫는 `**` 입력 시 | 볼드, 구문 숨김 |
| `*text*` | 닫는 `*` 입력 시 | 이탤릭, 구문 숨김 |
| `` `code` `` | 닫는 `` ` `` 입력 시 | 인라인 코드, 구문 숨김 |
| `$formula$` | 닫는 `$` 입력 시 | 인라인 수식 렌더링 |
| `[text](url)` | 닫는 `)` 입력 시 | 하이퍼링크로 변환 |
| `![alt](url)` | 닫는 `)` 입력 시 | 이미지 렌더링 |

**인라인 수식 (`$`)**

`$` 입력으로 인라인 수식 편집을 시작하고, 두 번째 `$`로 닫으면 KaTeX로 렌더링한다. 포커스 시 소스가 보이고, 포커스 아웃 시 렌더링 결과만 보인다.

**멘션 (`@`)**

`@`를 입력하면 날짜, 페이지, 사람을 멘션하는 팝업이 나타난다. `@today` → 오늘 날짜 삽입, `@파일명` → 페이지 링크 삽입. Notion 사용자에게 익숙한 패턴이다.

**Ghost Text AI 제안**

AI가 다음 문장/단락을 회색 텍스트로 제안한다. Tab으로 수락, Esc으로 무시, 계속 타이핑하면 자동 소멸. Ghost Text는 설정에서 토글 가능하며, API 키가 설정되지 않으면 표시되지 않는다.

```
│ Baram은 Tauri 2.0 기반의 경량 마크다운 에디터이다.
│ ▏주요 특징으로는 WYSIWYG 편집, KaTeX 수식 지원,    ← 회색 Ghost Text
│                                              [Tab] 수락
```

### 4.2.2 Layer 2: Contextual — 맥락에 따라 나타남

Layer 2는 사용자의 현재 컨텍스트(선택, 호버, 커서 위치)에 따라 자동으로 나타나는 UI이다.

**플로팅 서식 툴바**

텍스트를 선택하면 선택 영역 위에 서식 툴바가 팝업으로 나타난다.

```
                    선택된 텍스트
            ┌───────────────────────────┐
            │ 이것은 선택된 텍스트입니다.   │
            └───────────────────────────┘
                         ↑
     ┌─────────────────────────────────────────┐
     │ B  I  U  S  </>  🔗  ==  ✨AI  ⋯       │
     │ 볼드 이탤릭 밑줄 취소선 코드 링크 하이라이트 AI 더보기│
     └─────────────────────────────────────────┘
```

각 버튼의 동작:
- **B/I/U/S**: 토글 (이미 적용된 서식이면 해제)
- **</>**: 인라인 코드 토글
- **🔗**: 링크 삽입/편집 인라인 입력 표시
- **==**: 하이라이트 토글 (Extension 활성화 시)
- **✨AI**: AI 퀵 액션 드롭다운
- **⋯**: 더보기 (위첨자, 아래첨자, 주석, 서식 해제)

AI✨ 버튼 드롭다운:
```
┌──────────────────────────┐
│ ✏️ 개선 (Improve)         │
│ 📏 줄이기 (Shorten)       │
│ 📖 늘리기 (Expand)        │
│ 🌐 번역 (Translate) ▸     │ → 한국어/영어/일본어/중국어
│ 🎭 톤 변경 (Tone) ▸       │ → 격식체/캐주얼/기술적/친근한
│ 💡 설명 (Explain)         │
│ ─────────────────────── │
│ ⌨️ 커스텀 지시...          │ → 자연어 입력창
└──────────────────────────┘
```

**블록 핸들 메뉴**

블록(문단, 헤딩, 코드 블록 등)의 왼쪽 여백에 마우스를 호버하면 블록 핸들이 나타난다.

```
       ⠿  │ # 제목 텍스트                    ← 드래그 핸들 (블록 재배치)
       ⋮  │                                  ← 메뉴 핸들 (클릭 시 드롭다운)
          │ 본문 텍스트입니다. 이 문단은...
```

⋮ 클릭 시 드롭다운:
```
┌─────────────────────────┐
│ 🔄 블록 타입 변환 ▸       │ → Paragraph / H1~H6 / Quote / Callout / Code
│ 📋 복제                  │
│ 🗑️ 삭제                  │
│ ─────────────────────── │
│ ⬆️ 위로 이동              │
│ ⬇️ 아래로 이동             │
│ ─────────────────────── │
│ 📎 블록 ID 복사           │   ← ((block-id)) 참조용
│ 🔗 블록 링크 복사          │
└─────────────────────────┘
```

⠿ 드래그: 블록을 드래그하여 다른 위치로 이동한다. 드래그 중 파란색 삽입 라인이 가능한 위치를 표시한다.

**테이블 호버 조작**

테이블 위에 마우스를 호버하면 행/열 추가 버튼이 나타난다.

```
              [+]                      ← 열 추가 버튼 (위쪽 호버)
┌───────────┬───────────┐
│ Header 1  │ Header 2  │
├───────────┼───────────┤
│ Cell 1    │ Cell 2    │ [+]          ← 행 추가 버튼 (우측 호버)
├───────────┼───────────┤
│ Cell 3    │ Cell 4    │
└───────────┴───────────┘
              ↕                         ← 열 경계선 드래그로 폭 조절
```

테이블 셀 내 특수 동작: Tab → 다음 셀, Shift+Tab → 이전 셀, Enter → 셀 내 줄바꿈, Ctrl+L → 현재 행 선택, Ctrl+E → 현재 셀 선택.

**이미지 호버 툴바**

이미지 위에 마우스를 호버하면 조작 툴바가 나타난다.

```
┌──────────────────────────────────┐
│                                  │
│          [이미지 렌더링]           │
│                                  │
│    ○─────────────────────────○   │  ← 모서리 드래그로 크기 조절
│                                  │
│  [🔍줌] [📋복사] [📁경로] [☁️업로드] [🗑️삭제]  │  ← 하단 호버 툴바
└──────────────────────────────────┘
```

이미지 삽입 5가지 방법: (1) 드래그 앤 드롭, (2) 클립보드 붙여넣기(Cmd+V), (3) 슬래시 커맨드 `/image`, (4) 마크다운 직접 입력 `![](url)`, (5) 단축키 Cmd+Shift+I.

**컨텍스트 메뉴 (우클릭) — 7종**

Typora의 요소 타입별 동적 메뉴를 계승한다. 커서 위치의 ProseMirror 노드 타입을 감지하여 적절한 메뉴를 표시한다.

| 유형 | 핵심 항목 |
|------|----------|
| **일반 텍스트** | Cut/Copy/Paste, Copy As Markdown, Copy As HTML, Paragraph ▸, Format ▸, Insert ▸ |
| **이미지** | Copy Image/Path, Reveal in Finder, Move/Upload/Delete, Zoom ▸ |
| **테이블** | Add/Delete Row/Column, Align ▸, Copy/Delete Table |
| **코드 블록** | Copy Code, Select All in Block, Change Language, Line Numbers |
| **수식 블록** | Copy Math Source, Copy As Image, Auto Numbering |
| **다이어그램** | Copy Source, Copy/Save As Image, Delete, Zoom ▸ |
| **사이드바 파일** | Open, New File/Folder, Rename, Move/Delete, Reveal in Finder |

모든 유형에 공통: Copy As Markdown, Copy As HTML Code.

**Inline Edit (Cmd+K)**

텍스트를 선택한 상태에서 Cmd+K를 누르면 인라인 AI 편집 패널이 선택 영역 아래에 나타난다.

```
│ ┌─────────────────────────────────────┐
│ │ 선택된 텍스트가 여기에 있습니다.        │  ← 선택 영역
│ └─────────────────────────────────────┘
│ ┌─────────────────────────────────────┐
│ │ ✨ 무엇을 할까요?                     │  ← 자연어 입력
│ │ > 한국어로 번역해줘                    │
│ └─────────────────────────────────────┘
│
│ (AI 처리 후 diff 표시)
│
│ ┌─────────────────────────────────────┐
│ │ - 선택된 텍스트가 여기에 있습니다.      │  ← 빨간색 (삭제)
│ │ + 선택한 텍스트가 여기에 있습니다.      │  ← 초록색 (추가)
│ └─────────────────────────────────────┘
│ [✓ 수락]  [✗ 거부]  [↻ 재생성]  [✏️ 수정]
```

커맨드 팔레트의 Cmd+K와 Inline Edit의 Cmd+K는 컨텍스트로 구분한다. 텍스트가 선택된 상태에서 Cmd+K → Inline Edit, 선택 없이 Cmd+K → 커맨드 팔레트.

### 4.2.3 Layer 3: Global — 앱 전체 호출

Layer 3은 단축키로 호출하는 앱 전체 범위의 UI이다.

**커맨드 팔레트 (Cmd+K)**

모든 기능의 단일 진입점이다.

```
┌──────────────────────────────────────────────┐
│ >  검색어 입력...                              │
├──────────────────────────────────────────────┤
│ 📌 최근 사용                                  │
│   Toggle Bold                          ⌘B    │
│   Insert Table                         ⌘T    │
│   Open Quick Switcher                  ⌘O    │
├──────────────────────────────────────────────┤
│ 🔍 전체 명령                                  │
│   File: Save                           ⌘S    │
│   File: Export to PDF                         │
│   View: Toggle Source Mode             ⌘/    │
│   View: Toggle Sidebar              ⌘⇧L     │
│   Insert: Heading 1                    ⌘1    │
│   AI: Summarize Document                     │
│   AI: Review Writing Quality                 │
│   Theme: Switch to Dark Mode                 │
│   Settings: Open Editor Settings             │
│   Ext: Toggle Wikilinks                      │
│   [플러그인명]: 명령어                         │
├──────────────────────────────────────────────┤
│ 💡 팁: "ai", "theme", "export" 등으로 필터링   │
└──────────────────────────────────────────────┘
```

설계 핵심: MRU(Most Recently Used) 정렬로 최근 사용 명령이 상단에 표시된다. 카테고리 접두사(File:, View:, AI:, Theme:, Ext: 등)로 필터링 가능하다. 단축키가 있으면 오른쪽에 표시하여 학습을 유도한다. 설정도 검색 가능하다("font" → "Settings: Font Size", "theme" → "Theme: Switch to Dark"). 플러그인 명령도 통합된다.

**Quick Switcher (Cmd+O)**

파일/헤딩/블록을 퍼지 검색하여 빠르게 이동한다.

```
┌──────────────────────────────────────────────┐
│ 🔍 파일명, #헤딩, ^블록 검색...                │
├──────────────────────────────────────────────┤
│ 📄 skill-a.md                 skills/         │
│ 📄 skill-b.md                 skills/         │
│ 📄 README.md                  docs/           │
│ ─────────────────────────────────────────── │
│ + "검색어"로 새 파일 만들기                      │
└──────────────────────────────────────────────┘
```

특수 검색 문법: `filename#heading` → 특정 파일의 특정 헤딩으로 바로 이동, `filename^block-id` → 특정 블록으로 이동, 결과가 없으면 "새 파일 만들기" 옵션 표시, 별칭(aliases) 매칭 지원.

**글로벌 검색 (Cmd+Shift+F)**

사이드바에 검색 패널이 열린다.

```
┌──────────────────────────┐
│ 🔍 검색어 입력...          │
│ [Aa] [.*] [W] [↕ 바꾸기]  │  ← 대소문자/정규식/단어단위/바꾸기
├──────────────────────────┤
│ 📄 skill-a.md (3건)       │
│   L12: ...관련 **키워드**... │
│   L45: ...다른 **매칭**...  │
│   L78: ...세 번째 **결과**...│
├──────────────────────────┤
│ 📄 README.md (1건)        │
│   L5: ...매칭 **컨텍스트**...│
└──────────────────────────┘
```

**AI Chat Panel (Cmd+Shift+A)**

우측 사이드바에 AI 대화 패널이 열린다. Part 6에서 상세히 설계한다.

**설정 (Cmd+,)**

구조화된 설정 패널이 모달로 열린다. §4.8에서 상세히 설계한다.

## 4.3 전체 레이아웃 설계

### 메인 윈도우 레이아웃

3-Column 가변 레이아웃을 사용한다.

```
┌───────────────────────────────────────────────────────────────┐
│ [macOS 시스템 메뉴바]                                          │
│ Baram | File | Edit | Insert | View | Go                      │
├──────────┬──────────────────────────────────────┬─────────────┤
│          │ [탭바] 📄 skill-a.md × | 📄 README × │             │
│ 좌 사이드 │ ─────────────────────────────────── │ 우 사이드    │
│ (토글)    │                                      │ (토글)      │
│          │                                      │             │
│ · 파일트리 │           메인 에디터 영역             │ · 백링크    │
│ · 아웃라인 │           (WYSIWYG)                  │ · 프로퍼티  │
│ · 검색    │                                      │ · AI 채팅   │
│ · 북마크  │           최대 폭: 720px              │             │
│          │           좌우 패딩: 32px              │             │
│          │                                      │             │
│ 기본 200px│                                      │ 기본 300px  │
│ 리사이즈  │                                      │ 리사이즈    │
├──────────┴──────────────────────────────────────┴─────────────┤
│ [상태바]  📄 main  │  Words: 1,234  │  Ln 42 Col 15           │
└───────────────────────────────────────────────────────────────┘
```

기본 상태: 좌 사이드바 숨김, 우 사이드바 숨김, 에디터만 표시(원칙 3: 최소 인터페이스). 좌 사이드바는 Cmd+Shift+L로 토글하며, 우 사이드바는 특정 기능 호출 시 자동 열림(백링크 클릭, AI 패널 호출 등).

### 좌 사이드바 4가지 모드

사이드바 상단에 4개 아이콘 탭이 있으며, 클릭하여 모드를 전환한다.

**파일 트리** — Vault의 폴더/파일 구조를 표시한다. 마크다운 파일(.md, .markdown)만 기본 표시하며, 이미지/기타 파일은 필터로 표시 가능하다. 드래그앤드롭으로 파일 이동, 자연 정렬(1, 2, 10), 외부 변경 자동 감지. 하단에 검색 필터, 새 폴더, 새 파일 버튼.

**아웃라인** — 현재 문서의 H1~H6을 계층 트리로 표시한다. 클릭 시 해당 위치로 스크롤, 현재 커서 위치에 해당하는 헤딩 자동 하이라이트, 상단 검색 필터로 특정 헤딩 빠르게 찾기.

**글로벌 검색** — Cmd+Shift+F로 직접 진입 가능. 대소문자/정규식/단어 단위 옵션, 파일별 매칭 수와 컨텍스트 미리보기, 결과 클릭 시 해당 파일의 해당 위치로 점프.

**북마크** — 파일, 폴더, 헤딩, 검색어, 블록을 북마크하여 빠르게 접근. 폴더 구조로 북마크를 조직할 수 있다.

### 우 사이드바 3가지 모드

**백링크** — 현재 파일을 링크하는 다른 파일 목록. Linked Mentions(명시적 링크)과 Unlinked Mentions(파일명이 언급되었으나 링크되지 않은 곳)을 구분하여 표시. 클릭 시 해당 파일로 이동.

**프로퍼티 (Properties)** — 현재 파일의 YAML frontmatter를 GUI 폼으로 편집. 태그는 칩 UI, 날짜는 데이트 피커, 선택 항목은 드롭다운으로 표시. Skills 편집 모드에서 핵심 기능이 된다.

**AI 채팅** — LLM과의 대화 패널. `@파일명`으로 컨텍스트 파일을 참조하고, "Apply to Editor" 버튼으로 AI 응답을 에디터에 적용. Part 6에서 상세 설계.

### 반응형 동작

| 창 크기 | 동작 |
|--------|------|
| 넓음 (>1400px) | 3-column 모두 표시 가능 |
| 중간 (900~1400px) | 사이드바 한쪽만, 에디터 최대폭 유지 |
| 좁음 (<900px) | 사이드바 자동 숨김, 에디터만 |
| 전체화면 (F11) | 메뉴바 숨김, 상태바 숨김, 에디터만 |
| Zen Mode | 메뉴바 + 사이드바 + 상태바 + 탭바 모두 숨김, 에디터 중앙 집중 |

사이드바 Floating 모드: 설정에서 활성화하면, 좌측 가장자리에 마우스를 호버할 때 사이드바가 오버레이로 나타났다가 마우스가 떠나면 사라진다.

### Workspace 프리셋

| 프리셋 | 좌 사이드바 | 에디터 | 우 사이드바 | 추가 |
|--------|-----------|--------|-----------|------|
| **글쓰기** (Writing) | 숨김 | 전체 화면 | 숨김 | Focus Mode ON |
| **Skills 편집** (Skills) | 파일 트리 | 에디터 | 프로퍼티 | Skills Extension ON |
| **리서치** (Research) | 파일 트리 | 에디터 | AI 채팅 + 백링크 | AI Extension ON |

프리셋은 커맨드 팔레트에서 "workspace" 검색으로 전환하고, Cmd+Alt+1/2/3 단축키로 직접 전환할 수도 있다. 사용자 커스텀 프리셋도 저장 가능하다.

| 전체 아이템 수 축소 | 상세 명령은 커맨드 팔레트에서 검색. 메뉴는 발견(discovery)용 |

## 4.5 상태바 설계 (미니멀)

### 레이아웃

```
┌──────────────────────────────────────────────────────────────┐

### Typora 대비 변경

| 요소 | Typora | Baram | 이유 |
|------|--------|-------|------|
| Git 브랜치 | ✗ | ✅ | 개발자 사용자에게 필수 정보 |
| 워드카운트 | ✅ | ✅ | 유지 |
| 행:열 | ✅ | ✅ | 유지 |

| 현재 테마 | ✅ | ✗ | 커맨드 팔레트에서 전환 |

### 워드카운트 상세 팝업

워드카운트 영역을 클릭하면 상세 통계 팝업이 나타난다.

```
┌───────────────────┐
│  📊 문서 통계       │
│ ─────────────── │
│  단어     1,234    │
│  글자     5,678    │
│  줄         89    │
│  읽기시간   6분     │
│ ─────────────── │
│  📐 선택 영역       │   ← 텍스트 선택 시에만 표시
│  단어        23    │
│  글자        98    │
└───────────────────┘
```

### Git 브랜치 표시

Git이 초기화된 Vault에서 현재 브랜치명을 표시한다. 클릭 시 브랜치 전환 드롭다운이 나타난다. Git이 초기화되지 않은 Vault에서는 이 영역이 표시되지 않는다. Git 상태에 따라 아이콘이 변한다: 클린(📄), 변경 있음(📝), 동기화 필요(🔄).

## 4.7 테마 시스템 설계

### CSS Variables 기반 테마 구조

모든 시각적 속성은 CSS Variables로 정의되며, 테마 전환은 변수 세트를 교체하는 것으로 구현된다.

```
테마 파일 구조:
themes/
├── light.css            ← 기본 라이트 테마
├── dark.css             ← 기본 다크 테마
├── sepia.css            ← 세피아 (따뜻한 톤)
└── custom/              ← 사용자 커스텀 테마
    └── my-theme.css
```

테마 CSS 파일은 CSS Variables 오버라이드만 포함한다.

```css
/* themes/sepia.css */
:root {
  --bg-primary: #f4f1ea;
  --bg-secondary: #ebe8e1;
  --text-primary: #433422;
  --text-secondary: #8a7b6b;
  --accent-primary: #c06014;
  /* ... */
}
```

### 기본 제공 테마

| 테마 | 설명 | 영감 |
|------|------|------|
| **Baram Light** | 밝고 깨끗한 기본 테마. 푸른 계열 액센트 | GitHub, Typora Github |
| **Baram Dark** | 어두운 배경, 눈의 피로 최소화 | VS Code Dark, Typora Night |
| **Sepia** | 따뜻한 세피아 톤. 오래 읽기에 적합 | Typora Gothic, 종이 질감 |

### 사용자 커스텀 테마

Typora와 동일하게 단일 CSS 파일로 커스텀 테마를 만들 수 있다. 테마 폴더(`~/.baram/themes/`)에 CSS 파일을 넣으면 자동 인식된다. 커맨드 팔레트에서 "Open Theme Folder"로 해당 폴더를 열 수 있다.

### Glassmorphism / Translucent 효과 옵션

macOS에서 사이드바와 커맨드 팔레트에 반투명 효과를 적용하는 옵션을 제공한다. Tauri의 macOS vibrancy API를 활용한다. 기본은 비활성이며, 설정에서 활성화한다. 성능 영향이 있으므로 저사양 기기에서는 자동 비활성.

### 시스템 테마 자동 동기화

`prefers-color-scheme` 미디어 쿼리를 감시하여 OS 다크 모드 전환 시 자동으로 테마를 전환한다. "System Auto" 모드에서는 라이트/다크 각각에 대응하는 테마를 별도 설정할 수 있다(예: 라이트 → Baram Light, 다크 → Sepia).

첫 실행 시 선택적으로 시작할 수 있는 간단한 투어. 3단계로 구성된다.

1단계 — 커맨드 팔레트: "Cmd+K를 눌러보세요" → 커맨드 팔레트가 열림 → "모든 기능을 여기서 검색할 수 있습니다"

2단계 — 슬래시 커맨드: "빈 줄에서 /를 입력해보세요" → 슬래시 메뉴가 열림 → "헤딩, 테이블, 코드 블록 등을 삽입할 수 있습니다"

3단계 — 위키링크: "[[를 입력해보세요" → 파일 검색 팝업이 열림 → "문서 간 연결을 만들 수 있습니다"

투어는 "다시 보지 않기" 체크박스로 비활성화 가능하며, 커맨드 팔레트에서 "Welcome Tour" 검색으로 다시 실행할 수 있다.
