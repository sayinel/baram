# §5.1 Toggle 개선 — Notion 스타일 UI + Behavior

## 배경

Notion의 토글 블록 UI/UX를 분석하여 Baram의 토글을 개선한다.

## Notion 핵심 동작 요약

| 상황 | Notion 동작 |
|------|------------|
| **Enter (접힌 상태, summary 끝)** | 같은 타입의 새 toggle sibling 생성 |
| **Enter (펼친 상태, summary 끝)** | toggle 안에 빈 paragraph 생성 (첫 번째 자식) |
| **Enter (body 안 빈 paragraph)** | toggle 밖으로 탈출 → 아래에 paragraph 생성 |
| **Backspace (빈 summary)** | toggle → paragraph로 변환, 자식은 sibling으로 풀림 |
| **ArrowDown (접힌 상태)** | 다음 sibling 블록으로 (자식 건너뜀) |
| **ArrowDown (펼친 상태)** | 첫 번째 자식 블록으로 |
| **Cmd+Enter** | 현재 toggle 열기/닫기 |
| **Toggle Heading** | heading에 `is_toggleable` 속성 → heading이 접기/펼치기 가능 |

---

## 1. UI 스타일 (Notion 미니멀 스타일 적용)

| 항목 | 현재 | 제안 |
|------|------|------|
| 배경/테두리 | 제거됨 | 유지 (없음) |
| Summary font-weight | `600` (bold) | `400` (일반) — Notion과 동일 |
| 화살표 크기 | `6×8px` | `5×8px` (Notion ~10px 너비 삼각형) |
| 화살표-텍스트 간격 | `1.6em` indicator 폭 | `1.2em` (더 타이트하게) |
| 본문 들여쓰기 | 없음 | `padding-left: 1.2em` (화살표 너비만큼, summary 텍스트 시작점에 정렬) |
| 회전 애니메이션 | `0.15s` | `0.2s ease` |
| Hover 피드백 | 색상만 변경 | 화살표 영역에 `border-radius: 3px` + 미세한 배경색 |
| 블록 padding | `0.25em 0.5em` | `0.1em 0` (최소화) |

## 2. Toggle Heading (새 기능)

별도 노드를 만들지 않고, **기존 heading에 `isToggleable` 속성 추가**:

```
heading.attrs.isToggleable: boolean (default: false)
```

- Heading 1~3에 토글 기능 부여 가능 (Notion과 동일하게 H1/H2/H3만)
- ToC에 정상 표시됨 (toggle list와 차별점)
- 슬래시 커맨드: `/toggle heading 1`, `/toggle heading 2`, `/toggle heading 3`
- heading의 NodeView에 조건부 화살표 렌더링

## 3. Enter 동작 (핵심 — 접힌/펼친 상태에 따라 분기)

```
summary 끝에서 Enter:
  ├─ 접힌 상태 → 아래에 새 toggle sibling 생성
  └─ 펼친 상태 → toggle 안에 빈 paragraph 첫 자식으로 삽입

body 안 빈 paragraph에서 Enter:
  └─ toggle 밖으로 탈출 → 아래에 paragraph sibling 생성

빈 summary에서 Backspace:
  └─ toggle → paragraph 변환 (자식은 sibling으로 풀림)
```

## 4. 키보드 단축키

| 단축키 | 동작 |
|--------|------|
| `Cmd+Enter` | 현재 toggle 열기/닫기 |
| `ArrowDown` (접힌 상태) | 다음 sibling으로 이동 (자식 건너뜀) |
| `ArrowDown` (펼친 상태) | 첫 번째 자식으로 이동 |
| `Tab` | toggle 전체를 위 블록 안으로 들여쓰기 |
| `Shift+Tab` (body 자식) | 자식을 toggle 밖으로 꺼냄 |

## 5. 구현 우선순위

| 순서 | 항목 | 난이도 |
|------|------|--------|
| **1** | UI 스타일 개선 (미니멀, 들여쓰기, hover) | 낮음 |
| **2** | Enter 동작 개선 (접힌/펼친 분기) | 중간 |
| **3** | ArrowDown 접힌 상태 스킵 | 중간 |
| **4** | Cmd+Enter 토글 단축키 | 낮음 |
| **5** | Toggle Heading (heading에 isToggleable 추가) | 높음 |
