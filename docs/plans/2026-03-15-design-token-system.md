# Design Token System — Baram

**Date:** 2026-03-15
**Branch:** `refactoring/design-tokens`
**Status:** Approved — Implementation Ready

---

## Overview

Migrate Baram's monolithic `App.css` (14,506 lines) to a modular, Figma-compatible design token system using a 3-level token hierarchy, Style Dictionary v4 (DTCG format), and systematic Tailwind utility migration.

**Primary goals:**
1. Centralized token management (Figma-like)
2. Full CSS file modularity (from 1 file → 8 files)
3. Systematic naming convention
4. Figma Tokens Studio compatibility (future Figma ↔ code sync)
5. Zero visual regression (all changes verified)

---

## Token Architecture

### 3-Level Hierarchy

```
Level 1: Primitive   →  Level 2: Semantic    →  Level 3: Component CSS
     JSON (DTCG)              JSON (DTCG)            직접 CSS 작성
  Style Dictionary         Style Dictionary           Tailwind 4
     로 생성                   로 생성
```

```css
/* Level 1 — Primitive (raw values, no meaning) */
--color-blue-500: #3b82f6;
--color-blue-600: #2563eb;
--color-gray-50:  #f9fafb;

/* Level 2 — Semantic (meaning, references primitives) */
--color-accent-default: var(--color-blue-500);
--color-accent-hover:   var(--color-blue-600);
--color-bg-default:     var(--color-gray-50);

/* Level 3 — Component (in CSS files, references semantic) */
.editor { background: var(--color-bg-default); }
```

### Dark Mode

Semantic tokens are overridden at `[data-theme="dark"]` — primitives never change.

```css
/* base: light */
--color-bg-default: var(--color-gray-50);

/* dark override */
[data-theme="dark"] {
  --color-bg-default: var(--color-gray-900);
}
```

## Directory Structure

### Token Source (JSON)

```
tokens/
├── primitive/
│   ├── color.json               # Full color scales (blue-50~950, gray-50~950, red, green, etc.)
│   ├── spacing.json             # 4px grid scale (1=4px, 2=8px, ...)
│   └── typography.json          # font-size, weight, line-height, font-family
├── semantic/
│   ├── color-light.json         # --color-bg-*, --color-text-*, --color-accent-*
│   ├── color-dark.json          # dark mode overrides (references same primitives)
│   └── component.json           # callout colors, mood colors, graph colors
└── style-dictionary.config.ts   # Build config → outputs to src/styles/generated/
```

### CSS Output

```
src/styles/
├── generated/                   # ⚠️ DO NOT EDIT — auto-generated by Style Dictionary
│   ├── primitives.css           # --color-blue-*, --space-*, etc. in @theme block
│   └── semantic.css             # --color-bg-*, --color-text-*, etc. + dark overrides
├── base.css                     # @import "tailwindcss", html/body reset
├── editor.css                   # .tiptap-* styles (~1,800 lines)
├── layout.css                   # activity bar, sidebar, panels, tab bar (~800 lines)
├── toolbar.css                  # floating toolbar, block handle (~400 lines)
├── file-tree.css                # file tree component (~400 lines)
├── settings.css                 # settings modal + all tabs (~1,500 lines)
├── dialogs.css                  # command palette, quick capture, other dialogs (~600 lines)
├── components.css               # remaining UI components (~2,000 lines)
└── index.css                    # @import orchestrator (replaces App.css)
```

`src/App.tsx` imports `./styles/index.css` instead of `./App.css`.

## Token Naming Convention

### Primitives

```
--color-{palette}-{step}
--color-blue-50, --color-blue-100, ..., --color-blue-950
--color-gray-50, ..., --color-gray-950
--color-red-*, --color-green-*, --color-yellow-*, --color-purple-*
--color-white, --color-black

--space-1: 4px, --space-2: 8px, --space-4: 16px, ...

--font-size-xs, --font-size-sm, --font-size-base, --font-size-lg, ...
--font-weight-normal, --font-weight-medium, --font-weight-bold
--line-height-tight, --line-height-normal, --line-height-relaxed
--font-family-sans, --font-family-mono, --font-family-editor
```

### Semantic

```
--color-bg-default        (기존: --color-bg-primary)
--color-bg-subtle         (기존: --color-bg-secondary)
--color-bg-panel          (기존: --color-bg-sidebar)
--color-bg-elevated       (기존: --color-bg-tertiary)

--color-text-primary      (유지)
--color-text-secondary    (유지)
--color-text-disabled     (기존: --color-text-muted)

--color-border-default    (기존: --color-border)
--color-border-subtle     (기존: --color-border-light)

--color-accent-default    (기존: --color-accent)
--color-accent-hover      (유지)

--color-editor-bg         (유지)
--color-editor-text       (유지)
--color-editor-selection  (유지)
--color-editor-cursor     (유지)
```

### Components

```
--sidebar-width           (기존: --spacing-sidebar)
--editor-padding          (기존: --spacing-editor-padding)
--font-family-editor      (기존: --font-editor)
```

## Migration Phases

### Phase 1 — Style Dictionary Setup (시각적 변화: 없음)
1. Install `style-dictionary` v4
2. Create `tokens/primitive/color.json` (full color scales)
3. Create `tokens/primitive/spacing.json`, `typography.json`
4. Create `tokens/semantic/color-light.json`, `color-dark.json`, `component.json`
5. Create `style-dictionary.config.ts` → output to `src/styles/generated/`
6. Add `tokens:build` script to `package.json`
7. Run build → verify `generated/primitives.css` and `generated/semantic.css`
8. **Verification**: `npm run build` passes, no visual change

### Phase 2 — CSS File Split (시각적 변화: 없음)
1. Create `src/styles/` directory
2. Extract App.css sections into 7 component CSS files
3. Create `src/styles/index.css` with `@import` chain
4. Update `src/App.tsx` to import `./styles/index.css`
5. Remove original `App.css`
6. **Verification**: `npm run build` + `npm test` + Playwright screenshot diff

### Phase 3 — Variable Rename (⚠️ 위험 구간)
1. Generate migration map: old name → new name (complete inventory)
2. Use ast-grep/sed for bulk replacement across all CSS files
3. Add aliases in `generated/semantic.css` for any missed references
4. **Verification**: CSS variable audit script (0 unresolved references) + screenshot diff + `migration-report.md` with explicit list of all changed names

### Phase 4 — Tailwind Utility Migration (⚠️ 위험 구간, 점진적)
1. Component by component, replace BEM classes with Tailwind utilities
2. New components: Tailwind-first
3. **Verification**: per-component screenshot comparison before/after

### Phase 5 — tokens.json + Figma Structure (시각적 변화: 없음)
1. Export `tokens.json` in Tokens Studio format (script)
2. Document Figma ↔ code sync workflow
3. Add `tokens:export` script

## Verification Protocol (모든 Phase 공통)

1. `npm run build` — 빌드 오류 없음
2. `npm test` — 기존 테스트 전부 통과
3. `npm run typecheck` — TypeScript 오류 없음
4. Playwright 스크린샷 비교 — 픽셀 단위 diff (주요 화면 10개)
5. CSS variable 감사 스크립트 — 미해결 `var()` 참조 0개
6. **Phase 3/4 한정**: `migration-report.md` — 변경된 항목 전체 목록

**Audit 사이클**: 각 Phase 완료 후 Leader가 최소 15회 감사 진행. 문제 발견 시 재검토 → 타당한 문제만 수정 → 재검증.
