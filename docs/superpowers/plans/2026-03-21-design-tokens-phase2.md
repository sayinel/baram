# Design Tokens Phase 2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the entire Baram product from partial token coverage (16 theme keys, 30 undefined vars, ~366 hardcoded colors) to a complete semantic token contract where every UI surface responds to theme changes.

**Architecture:** Expand W3C DTCG token source → regenerate CSS via Style Dictionary v5 → expand `ThemeColors` interface (16→25 keys) → migrate all CSS/component consumers to use generated variables → add CI enforcement. Phase 2-1 (contract) is sequential; Phase 2-2 (migration) is parallelizable by file.

**Tech Stack:** Style Dictionary v5, W3C DTCG JSON, TypeScript, CSS custom properties, Zustand (persist v10→v11), Vitest
