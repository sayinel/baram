# §5.9 Callout — 구현 노트

## Requirements (설계서에서 추출)
- Obsidian 호환 콜아웃 구문: `> [!type] title` + `> body`
- 타입: tip, info, warning, danger, note, abstract, todo, example, quote, bug, success, failure, question (13종)
- NodeView: 타입별 아이콘 + 색상 + 접기/펼치기 UI
- 접기: `> [!tip]-` (collapsed), `> [!tip]+` (explicit expanded)
- 슬래시 커맨드 `/callout`

## Dependencies
- Blockquote transformer (callout은 markdown에서 blockquote 구문)
- remark-parse는 `> [!type]`을 blockquote → paragraph으로 파싱
- NodeViewWrapper (React)

## Technical Challenges
- **mdast 충돌**: remark-parse는 callout을 "blockquote"로 파싱. blockquote transformer와 callout을 구분해야 함
  - 해결: md-to-pm.ts에서 blockquote 변환 전 `[!type]` 패턴 감지 → callout으로 분기
- **제목 저장**: title을 attribute로 저장 (inline formatting 미지원, Obsidian 동일)
- **접기 상태**: `-`/`+` suffix를 collapsed attribute로 매핑

## Edge Cases
- 빈 제목: `> [!tip]` (제목 없음)
- 빈 본문: `> [!tip] Title` (본문 없음)
- 중첩: 콜아웃 안에 blockquote, 리스트, 코드 등
- block ID: 콜아웃 내부 paragraph에 `^id` 가능
- 일반 blockquote과 구분: `> [!` 패턴 없으면 blockquote 유지

## PM Schema
```
callout: {
  content: "block+"
  group: "block"
  defining: true
  attrs: {
    type: { default: "info" }
    title: { default: "" }
    collapsed: { default: false }
  }
}
```
