# §5.1 토글 블록 (Toggle / Details) — 구현 노트

## Requirements (설계서에서 추출)
- HTML `<details>` / `<summary>` 패턴을 마크다운에서 지원
- 슬래시 커맨드 `/toggle`로 삽입
- NodeView — 삼각형 화살표 + 제목 + 접기/펼치기
- 마크다운 직렬화: `<details><summary>제목</summary>\n\n내용\n\n</details>`
- 중첩 가능 (토글 안에 토글)
- PM 스키마: `toggle { content: "paragraph block*", group: "block", attrs: { open: true } }`
  - 첫 번째 paragraph = 토글 제목 (summary)

## remark-parse 동작
remark-parse는 `<details>` HTML을 3개의 분리된 mdast 노드로 파싱:
1. `html` 노드: `<details>\n<summary>Title</summary>`
2. 중간 블록 노드들 (paragraph, list, etc.)
3. `html` 노드: `</details>`

따라서 md-to-pm에서 연속된 `html` + block* + `html` 패턴을 감지하여 toggle 노드로 합쳐야 함.

## Dependencies
- `src/pipeline/md-to-pm.ts` — `<details>` 감지 + toggle 노드 생성
- `src/pipeline/pm-to-md.ts` — toggle → `<details>` HTML 직렬화
- Callout과 유사 패턴 (manual serialization, NodeView)

## Technical Challenges
1. **mdast 패턴 매칭**: `html(<details>...)` + block* + `html(</details>)` 패턴을 convertBlockChildren에서 감지
2. **summary 추출**: 첫 번째 html 노드에서 `<summary>` 내용 파싱
3. **open 속성**: `<details open>` → open: true, `<details>` → open: false
4. **중첩**: 토글 안에 토글 — content에 toggle 자체도 포함 가능
5. **빈 토글**: summary만 있고 body 없는 경우

## Edge Cases
- `<details open>` vs `<details>` (open 속성)
- 빈 body (summary만 존재)
- summary 없는 details (default "Details" 제목)
- 중첩 토글
- 토글 안에 다양한 블록 (코드, 리스트, 수식 등)
- body에 block ID가 있는 경우
